# جاهزية «المحامي العالمي» للتقديم في OpenAI Build Week

تاريخ المراجعة: 18 يوليو/تموز 2026

## القرار الحالي

المشروع مناسب لمسار **Work & Productivity**. النسخة الحالية تطبيق عامل وليست مجرد فكرة أو صورة: فيها خادم MCP، واجهة متعددة اللغات، أدوات بحث قانوني، فلاتر إجرائية، قوالب عرائض، بيانات تجريبية موسومة بوضوح، واختبارات آلية.

المتبقي قبل التقديم ليس توسيع المحتوى القانوني، بل نشر النسخة، توثيق استخدامها الحقيقي مع GPT-5.6، تسجيل الفيديو، وربط الأدلة في Devpost.

## الخطة التنفيذية حتى الموعد النهائي

الموعد الرسمي هو **21 يوليو/تموز 2026 الساعة 5:00 مساءً بتوقيت المحيط الهادئ الصيفي**، أي **22 يوليو/تموز الساعة 3:00 صباحاً بتوقيت بغداد**. لا تنتظر الساعات الأخيرة؛ الموعد الداخلي الآمن المقترح هو **21 يوليو الساعة 10:00 مساءً بتوقيت بغداد**.

### 18 يوليو — تثبيت الحزمة

1. راجع الاسم الذي سيظهر كمالك للمشروع والبريد العام للخصوصية.
2. شغّل `npm ci && npm run check && npm test`.
3. لا تضف نصوص قوانين أو قرارات غير موثقة لمجرد تكبير العرض.

### 19 يوليو — GitHub والنشر

1. أنشئ مستودعاً باسم `al-muhami-global`.
2. ارفع جميع ملفات مجلد المشروع مع سجل commits مؤرخ.
3. انشر من المستودع عبر Render باستخدام `render.yaml` أو `Dockerfile`.
4. اختبر `/healthz` و`/demo` و`/mcp` عبر HTTPS.

### 20 يوليو — ChatGPT والفيديو

1. افتح Developer mode في ChatGPT وأنشئ App من رابط `/mcp`.
2. شغّل النسخة النهائية في محادثة GPT-5.6، وتأكد أن الرحلات في دليل الحكام تعمل.
3. نفّذ مراجعة أو تحسيناً نهائياً حقيقياً في جلسة Codex المؤهلة إذا لم يكن استخدام GPT-5.6 موثقاً بعد.
4. اكتب `/feedback` في جلسة Codex الأساسية واحفظ Session ID.
5. سجّل فيديو إنجليزياً بصوت، أقل من ثلاث دقائق، وارفعه Public/Unlisted بما يسمح للحكام بمشاهدته دون طلب صلاحية.

### 21 يوليو — Devpost والإرسال

1. الصق النص الإنجليزي الجاهز واملأ الروابط والمعرّفات.
2. اختبر الفيديو في نافذة خاصة.
3. اختبر رابط التطبيق والمستودع من جهاز أو نافذة غير مسجل فيها.
4. راجع أن كل أسماء الدعاوى التجريبية موسومة بأنها وهمية.
5. أرسل المشاركة قبل الساعة 10:00 مساءً بتوقيت بغداد واحتفظ بصورة تأكيد الإرسال.

git branch -M main

لا ترفع كلمات مرور أو مفاتيح API أو بيانات موكلين حقيقية. إذا كان المستودع خاصاً، امنح حسابات التحكيم المذكورة في القواعد الحالية صلاحية الوصول؛ أما المستودع العام فهو أبسط للتحكيم مع إبقاء البيانات التجريبية فقط.

## بوابات تمنع الإرسال قبل حلها

- لا تدّعِ استخدام GPT-5.6 ما لم تشغّل المشروع فعلياً في جلسة محددة بهذا النموذج وتوثّق ذلك.
- لا ترسل مع حقول `[HTTPS_URL]` أو `[YOUTUBE_URL]` أو `[CODEX_SESSION_ID]`.
- تأكد دورياً أن البريد المنشور مخصص للتواصل العام وأن التحقق بخطوتين مفعّل.
- لا تستخدم أسماء الأطراف الحقيقية أو ملفات عملاء أو قرارات غير مصرح بإعادة نشرها.
- اجعل التطبيق والمستودع والفيديو متاحة للحكام مجاناً حتى نهاية التحكيم.
